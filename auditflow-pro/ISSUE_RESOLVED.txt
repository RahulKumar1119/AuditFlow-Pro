================================================================================
THE PROBLEM
================================================================================

Applicant names were showing as "Unknown Applicant" or "-" instead of actual
names extracted from PDF documents.

================================================================================
ROOT CAUSE IDENTIFIED
================================================================================

The Step Functions state machine had a bug in the ProcessAllDocuments Map state:

The ReturnProcessedDocument Pass state was not explicitly returning the
complete document with extracted_data. This caused the Map state to lose the
extracted_data when collecting results.

Data Flow Before Fix:
  ExtractData → extracted_data: {...}
  ↓
  SaveDocumentMetadata → ResultPath: null (discards output)
  ↓
  ReturnProcessedDocument → **BUG: No OutputPath specified**
  ↓
  Map state collects results → **extracted_data is lost!**
  ↓
  ValidateDocuments receives documents WITHOUT extracted_data
  ↓
  Can't extract names → Golden record without name field
  ↓
  Reporter falls back to "Unknown Applicant"

================================================================================
THE FIX
================================================================================

File: auditflow-pro/backend/step_functions/state_machine.asl.json

Changed:
  "ReturnProcessedDocument": {
    "Type": "Pass",
    "End": true
  }

To:
  "ReturnProcessedDocument": {
    "Type": "Pass",
    "Comment": "Return the complete processed document with extracted_data",
    "OutputPath": "$",
    "End": true
  }

The OutputPath: "$" ensures the complete state (including extracted_data) is
returned to the Map state.

================================================================================
DATA FLOW AFTER FIX
================================================================================

ExtractData → extracted_data: {...}
  ↓
SaveDocumentMetadata → ResultPath: null
  ↓
ReturnProcessedDocument → **OutputPath: "$" returns complete document**
  ↓
Map state collects results → **extracted_data is preserved!**
  ↓
ValidateDocuments receives documents WITH extracted_data
  ↓
Extracts names from extracted_data
  ↓
Golden record generated WITH name field
  ↓
Reporter extracts applicant_name correctly
  ↓
DynamoDB stores audit record with applicant_name
  ↓
Frontend displays applicant name correctly ✅

DEPLOYMENT INSTRUCTIONS
================================================================================

1. Verify the fix:
   grep -A 3 "ReturnProcessedDocument" auditflow-pro/backend/step_functions/state_machine.asl.json

2. Get State Machine ARN:
   aws stepfunctions list-state-machines \
     --query 'stateMachines[?name==`AuditFlowDocumentProcessing`].stateMachineArn' \
     --output text

3. Get IAM Role ARN:
   aws iam list-roles \
     --query 'Roles[?RoleName==`AuditFlowStepFunctionsRole`].Arn' \
     --output text

4. Update the state machine:
   aws stepfunctions update-state-machine \
     --state-machine-arn <STATE_MACHINE_ARN> \
     --definition file://auditflow-pro/backend/step_functions/state_machine.asl.json \
     --role-arn <ROLE_ARN>

5. Verify deployment:
   aws stepfunctions describe-state-machine \
     --state-machine-arn <STATE_MACHINE_ARN> \
     --query 'definition' \
     --output text | jq '.States.ProcessAllDocuments.Iterator.States.ReturnProcessedDocument'
